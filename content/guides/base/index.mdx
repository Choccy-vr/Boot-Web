<Callout title="Who is this for?">
This guide is for participants in the **Based Off** category of Boot YSWS — where you create a custom OS using an existing OS as the foundation. Ubuntu 22.04 LTS Minimal is a great starting point.
</Callout>

## What is Ubuntu 22.04 LTS Minimal?

Ubuntu 22.04 LTS (Jammy Jellyfish) in its **minimal** form contains only the packages required to boot and run the system — no desktop, no office suite, no extras. This makes it an excellent base for a custom OS because:

- **Full control**: Nothing you didn't ask for
- **Long-term support**: Maintained until April 2027 (community), April 2032 (Ubuntu Pro)
- **Rich ecosystem**: Thousands of packages available via `apt`
- **Great documentation**: Massive community, years of answers online

## Part 1 — Setting Up Your Development Environment

We strongly recommend developing inside a **virtual machine**. VMs let you snapshot state, roll back mistakes, and test your ISO without rebooting your host.

### Install QEMU on your host

**Ubuntu / Debian:**
```bash
sudo apt install -y qemu-system-x86 qemu-utils ovmf
```

**Arch Linux:**
```bash
sudo pacman -S qemu-full edk2-ovmf
```

**macOS (Homebrew):**
```bash
brew install qemu
```

### Get the Ubuntu 22.04 Server ISO

Download the Ubuntu 22.04 LTS Server ISO from [ubuntu.com/download/server](https://ubuntu.com/download/server). The Server ISO boots into a minimal text-based installer with no desktop.

### Create a VM and install Ubuntu

```bash
# Create a 20 GB virtual disk image
qemu-img create -f qcow2 ubuntu-base.qcow2 20G

# Boot the installer (Linux host with KVM)
qemu-system-x86_64 \
  -m 2048 \
  -smp 2 \
  -drive file=ubuntu-base.qcow2,format=qcow2 \
  -cdrom ubuntu-22.04.5-live-server-amd64.iso \
  -boot d \
  -enable-kvm \
  -vga virtio \
  -net nic -net user
```

<Callout title="macOS / Windows hosts">
Remove `-enable-kvm` if your host does not support KVM. The VM will run without hardware acceleration — slower but functional.
</Callout>

During installation:
- Choose **Ubuntu Server (minimized)** when prompted for the base install type
- Set up a username and password
- Skip optional snap installs unless you specifically need them

### Boot your installed VM

```bash
qemu-system-x86_64 \
  -m 2048 \
  -smp 2 \
  -drive file=ubuntu-base.qcow2,format=qcow2 \
  -enable-kvm \
  -vga virtio \
  -net nic -net user
```

Once logged in, update the system:

```bash
sudo apt update && sudo apt full-upgrade -y
sudo apt autoremove -y
```

## Part 3 — Customizing Your OS

There are two main approaches. Pick the one that matches your style.

### Approach A — Cubic (Easiest, GUI-based)

[Cubic](https://github.com/PJ-Singh-001/Cubic) is a wizard-based tool that lets you chroot into an existing Ubuntu ISO and make changes through an interactive terminal session. It is the fastest path to a working custom image.

#### Install Cubic

```bash
sudo apt-add-repository ppa:cubic-wizard/release
sudo apt update
sudo apt install -y cubic
```

<Callout title="Desktop required">Cubic is a graphical application and requires a display. If you installed Ubuntu Server with no desktop, first install a minimal desktop environment (`sudo apt install -y ubuntu-desktop-minimal`) and reboot.</Callout>

#### Use Cubic

1. Launch Cubic: `cubic`
2. Create or select a **project directory** (e.g. `~/my-os`)
3. Point Cubic at the **original Ubuntu ISO** to use as your base
4. Cubic extracts the ISO and drops you into a **virtual chroot terminal**
5. Inside the terminal, make your changes:

```bash
# Remove packages you don't want
apt purge -y apport ubuntu-advantage-tools

# Add your custom packages
apt install -y htop neofetch tree nano

# Set your OS identity
cat > /etc/os-release << 'EOF'
NAME="My Custom OS"
VERSION="1.0"
ID=mycustomos
ID_LIKE=ubuntu
PRETTY_NAME="My Custom OS 1.0"
VERSION_ID="1.0"
SUPPORT_URL="https://github.com/yourusername/my-os"
BUG_REPORT_URL="https://github.com/yourusername/my-os/issues"
EOF

# Custom login banner
cat > /etc/motd << 'EOF'

  Welcome to My Custom OS!

EOF
```

6. Click **Next** in Cubic to configure the ISO name, version, and filename
7. Click **Generate** to build your custom ISO

Cubic outputs the ISO in your project directory.

### Approach B — live-build (CLI, reproducible)

`live-build` is the official Debian/Ubuntu toolchain for building live images from a set of configuration files. Everything is declared as files in a `config/` directory, which makes builds fully reproducible.

#### Install live-build

```bash
sudo apt install -y live-build
```

#### Initialise a project

```bash
mkdir ~/my-os && cd ~/my-os

lb config \
  --distribution jammy \
  --archive-areas "main restricted universe multiverse" \
  --mirror-bootstrap http://archive.ubuntu.com/ubuntu/ \
  --mirror-binary http://archive.ubuntu.com/ubuntu/ \
  --mirror-binary-security http://security.ubuntu.com/ubuntu/ \
  --binary-images iso-hybrid
```

This creates a `config/` directory. All customisation goes here.

#### Add custom packages

```bash
mkdir -p config/package-lists

cat > config/package-lists/custom.list.chroot << 'EOF'
htop
neofetch
nano
tree
EOF
```

#### Add a customisation hook

Hooks are shell scripts that run inside the chroot during the build. Name them with a numeric prefix to control execution order.

```bash
mkdir -p config/hooks/live

cat > config/hooks/live/0100-os-identity.hook.chroot << 'HOOK'
#!/bin/bash
set -e

cat > /etc/os-release << 'EOF'
NAME="My Custom OS"
VERSION="1.0"
ID=mycustomos
ID_LIKE=ubuntu
PRETTY_NAME="My Custom OS 1.0"
VERSION_ID="1.0"
EOF

echo "My Custom OS 1.0" > /etc/issue
echo "" >> /etc/issue
HOOK

chmod +x config/hooks/live/0100-os-identity.hook.chroot
```

#### Build the ISO

```bash
sudo lb build 2>&1 | tee build.log
```

The first build downloads packages from Ubuntu's mirrors (~500 MB). Expect 15–40 minutes depending on your internet speed and CPU. The finished image will be named `live-image-amd64.hybrid.iso`.

#### Rebuild after changes

```bash
# Keeps downloaded packages (faster)
sudo lb clean

# Removes everything including the package cache (full clean)
sudo lb clean --all
```

## Part 4 — Testing Your Custom ISO

### Boot it in QEMU

```bash
qemu-system-x86_64 \
  -m 2048 \
  -smp 2 \
  -cdrom live-image-amd64.hybrid.iso \
  -boot d \
  -enable-kvm \
  -vga virtio \
  -net nic -net user
```

Replace `live-image-amd64.hybrid.iso` with the actual path to your ISO. Cubic places its output in the project directory you selected.

### Verify the Based Off requirements

Before submitting, confirm your OS meets all three requirements:

**It must boot**
Your OS should reach a login prompt or desktop after booting the ISO.

**It must be original**
Check that your branding is in place:
```bash
cat /etc/os-release   # Should show YOUR OS name, not Ubuntu
cat /etc/issue        # Should show your custom banner
```

**It must be able to edit/create a `.txt` file**
```bash
echo "Hello, Boot YSWS!" > hello.txt
cat hello.txt
nano hello.txt        # If nano is installed
```

## Going Further

Once you have a working base, here are some directions to take it:

- **Desktop environment**: `sudo apt install ubuntu-desktop-minimal`
- **Custom Plymouth theme**: Place your theme in `/usr/share/plymouth/themes/` and set it with `update-alternatives`
- **Custom systemd services**: Add `.service` files to `/etc/systemd/system/` and enable them
- **Custom kernel**: Compile your own kernel (see [kernel.org](https://www.kernel.org/) and Ubuntu's kernel docs)
- **GUI installer**: Add [Calamares](https://calamares.io/) or Ubuntu's Ubiquity to your image
