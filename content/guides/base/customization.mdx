## OS Identity

Override the OS name, version, and metadata by replacing `/etc/os-release`:

```bash
cat > /etc/os-release << 'EOF'
NAME="My Custom OS"
VERSION="1.0"
ID=mycustomos
ID_LIKE=ubuntu
PRETTY_NAME="My Custom OS 1.0"
VERSION_ID="1.0"
HOME_URL="https://example.com"
SUPPORT_URL="https://example.com/support"
BUG_REPORT_URL="https://example.com/issues"
EOF
```

Set the system hostname:

```bash
echo "mycustomos" > /etc/hostname
```

Customise the pre-login banner (shown at the login prompt on every TTY):

```bash
cat > /etc/issue << 'EOF'
  Welcome to My Custom OS 1.0  \l

EOF
```

A **TTY** (TeleTYpewriter) is a text-only terminal session. Linux runs several virtual terminals in the background — you can usually switch between them with Ctrl+Alt+F1 through F6. The `\l` token in `/etc/issue` is replaced by the current TTY name (e.g. `tty1`) at runtime.

## Plymouth Boot Splash

**Plymouth** is the program that shows the animated loading screen when your OS boots up — the logo or spinner you see before the desktop appears. It runs very early in the boot process, even before the desktop is ready.

It launches during the **initramfs** phase — a tiny, temporary filesystem that loads into RAM at the very start of boot. Think of the initramfs as a setup crew: it loads essential drivers and mounts your real filesystem, then hands off to the main OS. Plymouth hooks into this phase so you see something nice instead of a wall of text.

### Install Plymouth

```bash
apt install -y plymouth plymouth-themes
```

### List available themes

```bash
plymouth-set-default-theme --list
```

### Apply a built-in theme

```bash
# The -R flag rebuilds the initramfs so the theme is active on next boot
plymouth-set-default-theme -R spinner
```

Built-in themes include `spinner`, `bgrt`, `fade-in`, `solar`, and `tribar`.

### Create a custom Plymouth theme

```bash
# Create your theme directory
mkdir -p /usr/share/plymouth/themes/my-os

# Theme descriptor
tee /usr/share/plymouth/themes/my-os/my-os.plymouth << 'EOF'
[Plymouth Theme]
Name=My OS
Description=Custom boot theme for My Custom OS
ModuleName=script

[script]
ImageDir=/usr/share/plymouth/themes/my-os
ScriptFile=/usr/share/plymouth/themes/my-os/my-os.script
EOF

# Place your logo (PNG, recommended 200×200 px)
cp logo.png /usr/share/plymouth/themes/my-os/

# Minimal script — centres the logo on screen
tee /usr/share/plymouth/themes/my-os/my-os.script << 'EOF'
logo.image = Image("logo.png");
logo.sprite = Sprite(logo.image);
logo.sprite.SetX(Window.GetWidth()  / 2 - logo.image.GetWidth()  / 2);
logo.sprite.SetY(Window.GetHeight() / 2 - logo.image.GetHeight() / 2);
EOF

# Activate and rebuild initramfs
plymouth-set-default-theme -R my-os
```

<Callout title="Rebuilding the initramfs">
Plymouth themes must be embedded in the initramfs (the early-boot environment loaded into RAM before the main OS starts). Always run `plymouth-set-default-theme -R <theme>` (or `update-initramfs -u`) after making theme changes — otherwise the live ISO will still show the old splash.
</Callout>

## Boot Sounds

### Startup sound via systemd

**systemd** is the init system — the very first process that starts when your OS boots. It manages all background services (called *units*) and controls the order in which they start. A **systemd service** is a config file (a `.service` unit) that tells systemd to run a specific program at a certain point during boot or when a certain condition is met.

Create a systemd service that plays a sound at the end of the boot sequence. Place your audio file inside the image first:

```bash
mkdir -p /usr/share/sounds/my-os
cp boot.wav /usr/share/sounds/my-os/
```

Create the service unit:

```bash
tee /etc/systemd/system/boot-sound.service << 'EOF'
[Unit]
Description=Play boot sound
After=sound.target
Wants=sound.target

[Service]
Type=oneshot
ExecStart=/usr/bin/aplay /usr/share/sounds/my-os/boot.wav

[Install]
WantedBy=multi-user.target
EOF

systemctl enable boot-sound.service
```

<Callout title="Audio server dependency">
`aplay` uses ALSA directly. If your image ships PulseAudio, replace `ExecStart` with `/usr/bin/paplay /usr/share/sounds/my-os/boot.wav` and add `After=pulseaudio.service`. For PipeWire, use `pw-play` and `After=pipewire.service`.
</Callout>

### Desktop login sound via XDG autostart

**XDG autostart** is a cross-desktop standard that tells desktop environments (GNOME, Xfce, KDE, etc.) which apps to launch automatically when a user logs in. Any `.desktop` file placed in `/etc/xdg/autostart/` will run once per user session, as soon as the desktop loads — for every user on the system.

```bash
mkdir -p /etc/xdg/autostart

tee /etc/xdg/autostart/login-sound.desktop << 'EOF'
[Desktop Entry]
Type=Application
Name=Login Sound
Exec=paplay /usr/share/sounds/my-os/login.ogg
Terminal=false
X-GNOME-Autostart-enabled=true
EOF
```

This runs once per user session, immediately after the desktop loads.

## GRUB Boot Menu

**GRUB** (GNU GRand Unified Bootloader) is the program that runs immediately when your computer powers on, before the OS loads. It shows a menu where you can choose which OS or kernel to boot, then hands control over to it. Customising GRUB lets you show your OS name and a custom background at this very first screen.

Edit `/etc/default/grub` inside the chroot:

```bash
tee /etc/default/grub << 'EOF'
GRUB_DEFAULT=0
GRUB_TIMEOUT=5
GRUB_DISTRIBUTOR="My Custom OS"
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
GRUB_CMDLINE_LINUX=""
GRUB_BACKGROUND="/usr/share/grub/backgrounds/my-os.png"
EOF

# Regenerate grub.cfg
update-grub
```

Place a background image:

```bash
mkdir -p /usr/share/grub/backgrounds
cp background.png /usr/share/grub/backgrounds/my-os.png
```

<Callout title="Image format">
GRUB supports PNG, JPEG, and TGA. For best compatibility use a PNG at your target screen resolution (e.g. 1920×1080). Images are displayed unscaled if the resolution does not match.
</Callout>

## Custom Systemd Services

You can create your own systemd service to run any script automatically at boot. The `.service` file below tells systemd: "run this script once after the network is up, and restart it if it crashes."

```bash
# Place your script
cp my-script.sh /usr/local/bin/my-script.sh
chmod +x /usr/local/bin/my-script.sh

# Create the unit file
tee /etc/systemd/system/my-service.service << 'EOF'
[Unit]
Description=My Custom Startup Service
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/my-script.sh
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

systemctl enable my-service.service
```
